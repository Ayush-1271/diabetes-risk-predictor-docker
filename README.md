# Diabetes Progression Predictor - Flask ML API (Docker Deploy)

Project 2 of 2 for this assignment. A lightweight, accurate ML API that predicts a quantitative measure of diabetes progression one year after baseline, from 5 basic health-checkup values. Deployed on Render using Docker.

The non-Docker counterpart to this assignment (a separate ML task - Iris species classification) lives in its own repo: iris-flower-classifier.

## Dataset

Diabetes dataset (Efron, Hastie, Johnstone, Tibshirani, 2004) - built directly into scikit-learn (load_diabetes). 442 patients, 10 baseline variables originally, loaded here with scaled=False to get real-world units (age in years, actual blood pressure, actual glucose level) instead of scikit-learn's default pre-standardized z-scores - so the deployed form uses numbers an actual person could provide, not statistical encodings.

Chosen deliberately for the Docker-deployed project: it ships inside scikit-learn itself, so training needs no external download and is 100% reproducible anywhere (including network-restricted environments). It's also small enough that the trained model stays under 1MB while reaching a solid R-squared on held-out data.

## Features used

Reduced from the original 10 variables down to 5 that a general user could plausibly know from a basic checkup, dropping the detailed lipid panel (total cholesterol, LDL, HDL, cholesterol/HDL ratio, triglycerides) that requires lab results most people don't have on hand:

- age - age in years
- sex - 1 or 2 (as coded in the original clinical dataset)
- bmi - body mass index
- bp - average blood pressure (mm Hg)
- s6 - blood sugar / glucose level (mg/dL) - directly relevant to a diabetes predictor

Plus 2 engineered interaction features (bmi_bp_interaction, glucose_bmi_interaction) computed automatically inside the app - the user never needs to provide these.

## Project structure

- train.py - EDA, feature engineering, GridSearchCV, saves diabetes_model.pkl
- app.py - Flask app: web form ("/") and JSON API ("/api/predict")
- templates/index.html - Web UI
- diabetes_model.pkl - Trained scikit-learn pipeline (StandardScaler + RandomForestRegressor)
- requirements.txt - Python dependencies
- Dockerfile - Container build (this project's deploy path)
- .gitignore
- eda_*.png - EDA plots generated by train.py

## Run locally

Install dependencies:

    pip install -r requirements.txt

Retrain the model (regenerates diabetes_model.pkl and EDA plots):

    python train.py

Start the Flask dev server on http://localhost:5000:

    python app.py

## Run with Docker

    docker build -t diabetes-progression-api .
    docker run -p 8000:8000 diabetes-progression-api

Then visit http://localhost:8000

## API usage

Fields use real clinical units, not statistical encodings.

Example request:

    curl -X POST http://HOST/api/predict -H "Content-Type: application/json" -d "{\"age\": 45, \"sex\": 1, \"bmi\": 26.5, \"bp\": 92.0, \"s6\": 95}"

Example response:

    {
      "input": {"age": 45, "sex": 1, "bmi": 26.5, "bp": 92.0, "s6": 95},
      "prediction": {"predicted_disease_progression": 152.3}
    }

## Model details

- Algorithm: Random Forest Regressor (scikit-learn Pipeline with StandardScaler)
- Tuning: GridSearchCV, 5-fold cross-validation, over n_estimators, max_depth, min_samples_leaf
- Features: 5 real-world-unit baseline variables plus 2 engineered interaction features (bmi_bp_interaction, glucose_bmi_interaction)
- Evaluation: RMSE, MAE, R-squared (see eda_predicted_vs_actual.png)
